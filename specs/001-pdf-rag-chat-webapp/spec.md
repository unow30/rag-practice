# Feature Specification: PDF RAG 대화형 웹 앱

**Feature Name**: PDF RAG 대화형 웹 앱  
**Short Name**: pdf-rag-chat-webapp  
**Created**: 2026-04-18  
**Status**: Draft

---

## Overview

사용자가 PDF 문서를 업로드하면 시스템이 문서 내용을 추출·분석하여 벡터 데이터베이스에 저장하고, 사용자가 자연어로 질문할 수 있는 대화형 인터페이스를 제공한다. 시스템은 질문에 가장 관련성 높은 문서 구절을 검색(Retrieval-Augmented Generation)하여 근거 있는 답변을 생성한다.

- Q: 답변 표시 방식은? → A: 스트리밍 방식 (단어 단위 순차 표시)
- Q: 복수 문서에서 답변 생성 시 출처 표시 방식은? → A: 모든 관련 출처를 문서별로 구분하여 함께 표시

- Q: 검색 품질 평가 지표는? → A: Recall@k, Answerable@k, Exact/Partial Match, Latency를 평가 기준으로 사용
- Q: 개발 접근 방식은? → A: v0 단순 베이스라인 구축 후 평가 셋 기반 단계적 고도화

## Problem Statement

조직 내 PDF 문서(보고서, 매뉴얼, 계약서 등)는 분량이 방대하여 원하는 정보를 찾기 위해 전체를 읽어야 하는 비효율이 발생한다. 사용자는 특정 내용을 빠르게 검색하고 자연어로 질문하여 즉각적인 답변을 받기를 원한다.

---

## Goals

- PDF 문서에서 텍스트 내용을 정확하게 추출한다.
- 추출된 내용을 기반으로 자연어 질의응답(Q&A) 기능을 제공한다.
- 답변에 원문 출처(페이지 번호, 인용 구절)를 함께 제공하여 신뢰성을 높인다.
- 비기술 사용자도 쉽게 사용할 수 있는 직관적인 웹 인터페이스를 제공한다.

---

## Non-Goals

- PDF 이외의 문서 형식(Word, Excel, 이미지 등) 지원 (이후 확장 가능)
- 문서 편집·수정 기능
- 다국어 번역 기능
- 오프라인 모드 지원

## User Scenarios & Testing

### 시나리오 1: PDF 문서 업로드 및 분석

**Actor**: 일반 사용자  
**전제조건**: 사용자가 웹 앱에 접근할 수 있다.

1. 사용자가 웹 앱에 접속한다.
2. PDF 파일을 드래그 앤 드롭 또는 파일 선택 버튼으로 업로드한다.
3. 시스템이 업로드 진행 상황을 표시한다.
4. 시스템이 문서 처리 완료를 알린다.
5. 사용자가 대화창에서 질문을 입력할 수 있는 상태가 된다.

**성공 기준**: 업로드부터 질문 가능 상태까지 60초 이내에 완료된다.

---

### 시나리오 2: 자연어 질의응답

**Actor**: 일반 사용자  
**전제조건**: PDF 문서가 업로드 및 처리 완료된 상태.

1. 사용자가 대화창에 자연어로 질문을 입력한다. (예: "3분기 매출은 얼마인가요?")
2. 시스템이 관련 문서 구절을 검색한다.
3. 시스템이 답변과 함께 출처(페이지 번호, 인용 구절)를 표시한다.
4. 사용자가 추가 질문을 이어서 할 수 있다.

**성공 기준**: 질문 제출 후 5초 이내에 답변이 표시된다.

---

### 시나리오 3: 복수 문서 관리

**Actor**: 일반 사용자  
**전제조건**: 사용자가 이미 1개 이상의 문서를 업로드한 상태.

1. 사용자가 추가 PDF를 업로드한다.
2. 사용자가 특정 문서 또는 전체 문서를 대상으로 질문 범위를 선택한다.
3. 시스템이 선택된 범위 내에서 답변을 생성한다.

**성공 기준**: 여러 문서 중 원하는 문서만 선택하여 질의할 수 있다.

---

### 시나리오 4: 문서 삭제

**Actor**: 일반 사용자  
**전제조건**: 문서가 1개 이상 등록된 상태.

1. 사용자가 문서 목록에서 삭제할 문서를 선택한다.
2. 확인 팝업이 표시된다.
3. 사용자가 확인하면 문서와 관련 데이터가 삭제된다.

**성공 기준**: 삭제된 문서를 대상으로 한 질의에서 해당 문서 내용이 반환되지 않는다.

## Functional Requirements

### FR-01: PDF 업로드

- 사용자는 웹 인터페이스를 통해 PDF 파일을 업로드할 수 있어야 한다.
- 업로드 가능한 최대 파일 크기는 50MB이다.
- 파일 형식이 PDF가 아닐 경우 명확한 오류 메시지를 표시한다.
- 업로드 진행률이 시각적으로 표시되어야 한다.
- 동시에 최대 5개 파일을 일괄 업로드할 수 있다.

### FR-02: 문서 내용 추출

- 시스템은 업로드된 PDF에서 텍스트를 자동으로 추출한다.
- 처리 대상 PDF는 주로 **텍스트, 표(table), 그래프(chart/figure)** 가 혼합된 문서이며, 세 가지 콘텐츠 유형 모두 추출 대상이다.
- 텍스트가 직접 포함된 PDF를 우선 지원하며, 스캔 이미지 기반 PDF에 대한 OCR은 초기 범위에서 제외하고 이후 단계에서 선택적 기능으로 추가한다.
- 스캔 PDF를 업로드할 경우 "이미지 기반 PDF는 현재 지원되지 않습니다"라는 안내 메시지를 표시한다.
- 표와 그래프의 내용이 질의응답에 활용될 수 있도록 구조화된 형태로 저장된다.
- 추출된 텍스트는 페이지 번호 정보와 함께 저장된다.
- 처리 실패 시 사용자에게 실패 원인을 안내한다.

### FR-03: 벡터 인덱싱

- 추출된 텍스트는 의미 단위로 분할되어 검색 가능한 형태로 저장된다.
- 문서 처리 완료 후 즉시 질의가 가능해야 한다.

### FR-04: 대화형 질의응답

- 사용자는 자연어로 질문을 입력할 수 있다.
- 시스템은 업로드된 문서에서 관련 구절을 검색하여 답변을 생성한다.
- 답변에는 반드시 출처(문서명, 페이지 번호, 관련 인용문)가 포함되어야 한다.
- 시스템은 **문서에서 근거를 찾을 수 없는 내용은 추측하거나 생성하지 않는다.** 문서 내용으로 답변할 수 없는 질문에는 "문서에서 관련 정보를 찾을 수 없습니다"라고 안내한다.
- 표나 그래프에서 근거를 찾은 경우, 해당 내용을 텍스트로 설명하고 출처(페이지 및 위치)를 함께 제공한다.
- 답변은 생성되는 즉시 단어 단위로 순차적으로 표시된다(스트리밍). 사용자는 답변이 완성되기 전에도 내용을 확인할 수 있다.
- 복수 문서에서 근거를 찾은 경우, 출처를 문서별로 구분하여 모두 표시한다.
- AI 서비스가 30초 이내에 응답하지 않으면 오류 메시지를 표시하고 재시도 버튼을 제공한다.
- 대화 기록이 세션 내에서 유지되어 이전 맥락을 참고한 후속 질문이 가능하다.

### FR-05: 문서 관리

- 사용자는 업로드된 문서 목록을 확인할 수 있다.
- 각 문서에 대해 이름, 업로드 일시, 페이지 수, 처리 상태를 확인할 수 있다.
- 사용자는 개별 문서를 삭제할 수 있다.
- 여러 문서가 있는 경우 질의 대상 문서를 선택/전체 선택할 수 있다.
- 저장 가능한 문서는 최대 20개이며, 한도 초과 시 "최대 문서 수(20개)에 도달했습니다. 기존 문서를 삭제 후 업로드해 주세요"라는 안내 메시지를 표시한다.

### FR-06: 접근 방식

- 이 시스템은 인증 없이 단일 사용자가 사용하는 개인용 앱으로 제공된다.
- 로그인, 회원가입, 비밀번호 관리 기능은 범위에 포함되지 않는다.
- 앱에 접속하면 별도의 인증 절차 없이 바로 문서 업로드 및 질의 기능을 사용할 수 있다.

### FR-07: 대화 기록

- 세션 내 대화 기록이 화면에 시각적으로 표시된다.
- 대화 기록은 브라우저 탭(세션) 단위로 유지되며, 탭을 닫거나 재접속하면 초기화된다.
- 업로드된 문서는 세션과 무관하게 영구 저장되어 재접속 후에도 동일하게 사용할 수 있다.
- 사용자는 세션 내에서 대화 기록을 수동으로 초기화할 수 있다.

## Success Criteria

| 기준 | 측정 방법 | 목표값 |
|------|-----------|--------|
| 문서 처리 시간 | 업로드 완료 후 질의 가능 상태까지 | 20페이지 기준 30초 이내 |
| 질의 응답 시간 (첫 토큰) | 질문 제출 후 첫 단어 표시까지 | 3초 이내 |
| AI 서비스 장애 감지 | 타임아웃 후 오류 안내까지 | 30초 이내 |
| 텍스트 추출 정확도 | 표준 PDF 10종 테스트 | 95% 이상 |
| 검색 재현율 (Recall@k) | 정답 청크가 상위 k개 결과 안에 포함되는 비율 | 상위 5개 결과 기준 80% 이상 |
| 답변 가능성 (Answerable@k) | 상위 k개 근거만으로 정답 생성 가능한 비율 | 테스트 질문 셋 기준 80% 이상 |
| 답변 정확도 (Exact/Partial Match) | 최종 답변이 정답과 일치하는 비율 | 완전 일치 60% 이상, 부분 일치 포함 시 85% 이상 |
| 최대 문서 크기 | 처리 가능한 단일 파일 크기 | 50MB |
| 동시 사용 지원 | 단일 사용자 기준 동시 처리 요청 수 | 문서 처리와 질의가 동시에 진행되어도 응답 시간 기준 유지 |
| 대화 연속성 | 세션 내 이전 맥락 참조 질문 성공률 | 90% 이상 |

## Key Entities

### Document (문서)

| 속성 | 설명 |
|------|------|
| ID | 고유 식별자 |
| name | 파일명 |
| uploaded_at | 업로드 일시 |
| page_count | 페이지 수 |
| status | 처리 상태 (대기/처리중/완료/실패) |
| size_bytes | 파일 크기 |

### Chunk (문서 조각)

| 속성 | 설명 |
|------|------|
| ID | 고유 식별자 |
| document_id | 소속 문서 ID |
| content | 텍스트 내용 |
| page_number | 원문 페이지 번호 |
| vector | 의미 벡터 표현 |

### Conversation (대화)

| 속성 | 설명 |
|------|------|
| ID | 고유 식별자 |
| messages | 질문/답변 메시지 목록 |
| created_at | 대화 시작 일시 |
| document_scope | 질의 대상 문서 목록 |

### Message (메시지)

| 속성 | 설명 |
|------|------|
| ID | 고유 식별자 |
| role | 역할 (user/assistant) |
| content | 메시지 내용 |
| sources | 답변 근거 출처 목록 |
| created_at | 생성 일시 |

## Assumptions

- 사용자는 현대 웹 브라우저(Chrome, Firefox, Safari, Edge 최신 버전)를 사용한다고 가정한다.
- 기본적으로 텍스트 기반 PDF를 주요 대상으로 하며, OCR은 선택적 기능으로 처리한다.
- 사용자는 한국어로 질문하고 한국어 답변을 기대한다고 가정한다 (다국어 혼용 문서도 지원).
- 업로드된 문서는 사용자의 세션이 종료되어도 지속 저장된다고 가정한다.
- 시스템은 인터넷에 연결된 환경에서 동작한다.
- 인증 없는 단일 사용자 환경을 기본으로 가정한다. 멀티유저 지원은 이후 확장 시 고려한다.
- 시스템은 단순 기본 파이프라인(v0 베이스라인)을 먼저 구축하고, 평가 결과를 기반으로 검색·추출 품질을 단계적으로 고도화하는 방식으로 개발된다.
- 품질 평가를 위해 10~20개 이상의 질문-정답 평가 셋을 사전에 구성하여 반복 측정한다.

---

## Dependencies

- AI 언어 모델 서비스 (답변 생성용)
- 벡터 검색 저장소
- PDF 텍스트 추출 라이브러리
- (선택) OCR 서비스

---

## Out of Scope

- 모바일 앱(iOS/Android) 제공
- PDF 이외 문서 형식 지원
- 실시간 협업(다중 사용자 동시 편집)
- 문서 버전 관리
- 답변 내보내기/공유 기능 (이후 단계에서 고려 가능)
